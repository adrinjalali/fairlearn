fairlearn.metrics package¶

Functionality for computing metrics, with a particular focus on group metrics.

For our purpose, a metric is a function with signature f(y_true, y_pred, ....) where y_true are the set of true values and y_pred are values predicted by a machine learning algorithm. Other arguments may be present (most often sample weights), which will affect how the metric is calculated.

The group metrics in this module have signatures g(y_true, y_pred, group_membership, ...) where group_membership is an array of values indicating a group to which each pair of true and predicted values belong. The metric is evaluated for the entire set of data, and also for each subgroup identified in group_membership.

class fairlearn.metrics.GroupMetricResult¶

Bases: object

Class to hold the result of a grouped metric.

Grouped metrics are produced by the metric_by_group() function.

property argmax_set¶

Return the set of groups corresponding to the minimum.

This is only set if the metric is a scalar, and will be a set of keys to tbe by_group dictionary.

property argmin_set¶

Return the set of groups corresponding to the minimum.

This is only set if the metric is a scalar, and will be a set of keys to tbe by_group dictionary.

property by_group¶

Return the metric calculated for each sub-group in the dataset.

This is a dictionary whose keys are the unique members of the group_membership data. The corresponding values are the result of applying the metric function to the set of y_true and y_pred entries for each key.

property maximum¶

Return the maximum value of the metric in the by_group dictionary.

This is only set if the metric is a scalar.

property minimum¶

Return the minimum value of the metric in the by_group dictionary.

This is only set if the metric is a scalar.

property overall¶

Return the metric calculated over the entire dataset.

property range¶

Return the value of maximum-minimum.

This is only set if the metric is a scalar.

property range_ratio¶

Return the value of minimum/maximum.

This is only set if the metric is a scalar.

fairlearn.metrics.balanced_root_mean_squared_error(y_true, y_pred, sample_weight=None)¶

Calculate the mean of the root mean squared error (RMSE) for the positive and negative cases.

Used for binary logistic regression, this computes the error as

\[\frac{\text{RMSE}(Y=0) + \text{RMSE}(Y=1)}{2}\]

The classes are constrained to be \(\in {0, 1}\). The y_true values must always be one of these, while y_pred can be a continuous probability (which could be thresholded to get a predicted class).

Internally, this builds on the sklearn.metrics.mean_squared_error() routine.

fairlearn.metrics.fallout_rate(y_true, y_pred, sample_weight=None)¶

Calculate the fallout rate (also called the False Positive Rate).

At the present time, this routine only supports binary classifiers with labels \(\in {0, 1}\). By definition, this is the complement of the Specificity, and so uses specificity_score() in its calculation.

fairlearn.metrics.mean_prediction(y_true, y_pred, sample_weight=None)¶

Calculate the (weighted) mean prediction.

The true values are ignored, but required as an argument in order to maintain a consistent interface

fairlearn.metrics.mean_overprediction(y_true, y_pred, sample_weight=None)¶

Calculate the (weighted) mean overprediction.

This is the (weighted) mean of the error where any negative errors (i.e. underpredictions) are set to zero

fairlearn.metrics.mean_underprediction(y_true, y_pred, sample_weight=None)¶

Calculate the (weighted) mean underprediction.

This is the (weighted) mean of the error where any positive errors (i.e. overpredictions) are set to zero. The absolute value of the underpredictions is used, so the returned value is always positive.

fairlearn.metrics.miss_rate(y_true, y_pred, sample_weight=None)¶

Calculate the miss rate (also called the False Negative Rate).

At the present time, this routine only supports binary classifiers with labels \(\in {0, 1}\). By definition, this is the complement of the True Positive Rate, so this routine uses the sklearn.metrics.recall_score() routine.

fairlearn.metrics.selection_rate(y_true, y_pred, *, pos_label=1, sample_weight=None)¶

Calculate the fraction of predicted labels matching the ‘good’ outcome.

The argument pos_label specifies the ‘good’ outcome.

fairlearn.metrics.specificity_score(y_true, y_pred, sample_weight=None)¶

Calculate the specificity score (also called the True Negative Rate).

At the present time, this routine only supports binary classifiers with labels \(\in {0, 1}\). The calculation uses the sklearn.metrics.confusion_matrix() routine.
