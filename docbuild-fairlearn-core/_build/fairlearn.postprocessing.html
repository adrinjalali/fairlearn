fairlearn.postprocessing package¶

This module contains methods which operate on a predictor, rather than an estimator.

The predictor’s output is adjusted to fulfill specified parity constraints. The postprocessors learn how to adjust the predictor’s output from the training data.

class fairlearn.postprocessing.PostProcessing(*, unconstrained_predictor=None, estimator=None, constraints=None)¶

Bases: object

Abstract base class for postprocessing approaches for disparity mitigation.

Parameters

  • unconstrained_predictor – A predictor with a predict(X) method that has already been trained on the training data; the predictor will subsequently be used in the mitigator for unconstrained predictions; can only be specified if estimator is None

  • estimator (estimator) – An estimator implementing fit(X, y) and predict(X) methods that will be trained on the training data and subsequently used in the mitigator for unconstrained predictions; can only be specified if unconstrainted_predictor is None

  • constraints (str) – the parity constraints to be enforced represented as a string

Fits the model.

The fit is based on training features and labels, sensitive features, as well as the fairness-unaware predictor or estimator. If an estimator was passed in the constructor this fit method will call fit(X, y, **kwargs) on said estimator.

class fairlearn.postprocessing.ThresholdOptimizer(*, unconstrained_predictor=None, estimator=None, constraints='demographic_parity', grid_size=1000, flip=True, plot=False)¶

Bases: fairlearn.postprocessing.PostProcessing

An Estimator based on the threshold optimization approach.

The procedure followed is described in detail in Hardt et al. (2016).

Parameters

  • unconstrained_predictor (A trained predictor) – The trained predictor whose output will be post processed

  • estimator (An untrained estimator) – An untrained estimator that will be trained, and subsequently its output will be post processed

  • grid_size (int) – The number of ticks on the grid over which we evaluate the curves. A large grid_size means that we approximate the actual curve, so it increases the chance of being very close to the actual best solution.

  • flip (bool) – Allow flipping to negative weights if it improves accuracy.

  • plot (bool) – Show ROC/selection-error plot if True
